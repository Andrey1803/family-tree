# 📸 ГИБРИДНОЕ ХРАНЕНИЕ ФОТО

## 🎯 Архитектура

```
┌─────────────────────────────────────────────────────────┐
│  Desktop-версия (Windows)                               │
│  ┌─────────────────────────────────────────────────┐   │
│  │  Фото (оригинал) → Сжатие → photo_full (800px)  │   │
│  │                              ↓                   │   │
│  │                       Миниатюра (200px) → photo  │   │
│  │                              ↓                   │   │
│  │  Локально: photos/username/person_id.jpg         │   │
│  └─────────────────────────────────────────────────┘   │
│                         ↓ синхронизация                │
└─────────────────────────────────────────────────────────┘
                         ↓
┌─────────────────────────────────────────────────────────┐
│  Сервер синхронизации (Railway)                         │
│  ┌─────────────────────────────────────────────────┐   │
│  │  SQLite БД:                                     │   │
│  │  - photo: BLOB (миниатюра 200px, ~10-30 KB)     │   │
│  │  - photo_full: BLOB (полное 800px, ~50-150 KB)  │   │
│  │  - photo_path: TEXT (путь для desktop)          │   │
│  └─────────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────────┘
                         ↓
┌─────────────────────────────────────────────────────────┐
│  Веб-версия (браузер)                                   │
│  ┌─────────────────────────────────────────────────┐   │
│  │  Карточка персоны: /api/photo/<id> (миниатюра)  │   │
│  │  Просмотр: /api/photo/<id>/full (полное фото)   │   │
│  └─────────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────────┘
```

---

## 📊 Размеры фото

| Тип | Размер | Качество | Размер файла |
|-----|--------|----------|--------------|
| **Миниатюра** | 200×200 | 75% | ~10-30 KB |
| **Полное** | 800×800 | 85% | ~50-150 KB |
| **Оригинал** | любой | 100% | сохраняется локально |

---

## 💾 Хранение данных

### Desktop-версия
- **Оригинал фото**: `photos/<username>/<person_id>.jpg`
- **photo_path**: относительный путь к файлу
- **photo**: base64 миниатюра (для синхронизации)
- **photo_full**: base64 полное фото (для веба)

### Сервер синхронизации
- **photo**: BLOB (миниатюра base64)
- **photo_full**: BLOB (полное фото base64)
- **photo_path**: TEXT (путь, не используется в вебе)

### Веб-версия
- **photo**: загружается с сервера для карточек
- **photo_full**: загружается с сервера при просмотре

---

## 🔄 Процесс синхронизации

### 1. Добавление фото в desktop-версии
```
Пользователь выбирает файл → Сжатие → Создание миниатюры →
Сохранение в model.person[id].photo и model.person[id].photo_full →
Синхронизация с сервером
```

### 2. Синхронизация с сервером
```
Desktop отправляет JSON с persons →
Сервер извлекает photo и photo_full (base64) →
Сохраняет в SQLite как BLOB
```

### 3. Загрузка в веб-версии
```
Веб загружает JSON с сервера →
photo и photo_full уже в данных →
Отображение через /api/photo/<id> или /api/photo/<id>/full
```

---

## 🛠 API Endpoints

### Миниатюра (для карточек)
```
GET /api/photo/<person_id>
Ответ: изображение (image/jpeg или image/png)
```

### Полное фото (для просмотра)
```
GET /api/photo/<person_id>/full
Ответ: изображение (image/jpeg или image/png)
```

## 📈 Расчёт места на сервере

### Формула
```
Размер дерева = (N персон × 30 KB) + (N персон × 100 KB)
              = N × 130 KB
```

### Примеры
| Персон | Размер БД | Тариф Railway |
|--------|-----------|---------------|
| 42 | ~5.5 MB | Hobby ($5) ✅ |
| 100 | ~13 MB | Hobby ($5) ✅ |
| 500 | ~65 MB | Hobby ($5) ✅ |
| 1000 | ~130 MB | Hobby ($5) ✅ |
| 5000 | ~650 MB | Pro ($20) ✅ |

**Вывод:** Тарифа **Hobby ($5/мес)** достаточно для **~4000 персон** с фото.

## 🔧 Миграция существующих данных

### Если фото уже сохранены в desktop-версии:
1. Откройте персону в режиме редактирования
2. Нажмите "Выбрать фото" заново (или просто "Сохранить")
3. Фото будет перекодировано в новый формат

### Если фото уже на сервере:
1. Сервер автоматически добавит поле `photo_full`
2. Старые фото останутся в поле `photo`
3. Новые фото сохранятся в оба поля

---

## 🐛 Устранение проблем

### Фото не отображается в вебе
1. Проверьте, что фото сохранено в desktop-версии
2. Выполните синхронизацию
3. Проверьте, что `photo_full` не пустой

### Фото слишком большое
1. Уменьшите `PREVIEW_SIZE` в `photo_service.py`
2. Уменьшите `quality` при сжатии

### Ошибка "PIL не доступен"
```bash
pip install Pillow
```

## 📝 Рекомендации

1. **Перед загрузкой**: сжимайте фото до 1920×1080
2. **Формат**: используйте JPEG вместо PNG
3. **Синхронизация**: выполняйте после каждого добавления фото
4. **Резервная копия**: храните оригиналы фото локально
