# 🔧 ИСПРАВЛЕНИЕ ОШИБКИ "Failed to fetch" В БРАУЗЕРЕ

## ❌ ПРОБЛЕМА

В консоли браузера ошибка:
```
[SAVE] ❌ Network error: TypeError: Failed to fetch
```

**API работает** (проверено скриптами), но **браузер не может отправить запрос**.

---

## 🔍 ВОЗМОЖНЫЕ ПРИЧИНЫ

### 1. Смешанный контент (HTTP → HTTPS)

**Проблема:** Сайт на HTTPS, но запросы идут на HTTP (или наоборот).

**Проверка:**
- Откройте сайт: `https://family-tree-production-0e7d.up.railway.app`
- Посмотрите в адресной строке — есть ли значок 🔒 (замкА)

**Решение:**
- Убедитесь что используете **HTTPS** (не HTTP!)
- Если перешли по HTTP — Railway должен перенаправить на HTTPS

---

### 2. Блокировщики рекламы/трекеров

**Проблема:** uBlock Origin, AdBlock, Privacy Badger блокируют запросы.

**Проверка:**
- Откройте консоль (F12) → вкладка **Console**
- Посмотрите есть ли ошибки CORS или блокировки

**Решение:**
1. Отключите блокировщики для этого сайта
2. Или добавьте в исключения: `family-tree-production-0e7d.up.railway.app`

---

### 3. Проблемы с куками (SameSite)

**Проблема:** Куки сессии не отправляются.

**Проверка в браузере:**
1. F12 → **Application** → **Cookies**
2. Есть ли куки `session`?

**Решение:**
- Очистите кук/и и кэш (Ctrl+Shift+Delete)
- Попробуйте режим инкогнито (Ctrl+Shift+N)

---

### 4. CORS (Cross-Origin)

**Проблема:** Запросы блокируются политикой CORS.

**Проверка:**
- F12 → Console → ищите `CORS policy`

**Решение:**
- Убедитесь что запросы идут на **тот же домен** (не localhost!)

---

### 5. Брандмауэр/Антивирус

**Проблема:** ESET, Касперский блокируют запросы.

**Проверка:**
- Попробуйте с другого устройства/сети
- Отключите антивирус на время теста

**Решение:**
- Добавьте сайт в исключения антивируса

---

## 🛠️ ДИАГНОСТИКА

### Шаг 1: Проверка доступности

Откройте в браузере:
```
https://family-tree-production-0e7d.up.railway.app/api/check-session
```

**Ожидается:** `{"has_server_token":false,"is_admin":false,"username":null,...}`

**Если 404 или ошибка:** Сервер недоступен или проблема с HTTPS.

---

### Шаг 2: Проверка входа

1. Откройте: `https://family-tree-production-0e7d.up.railway.app/login`
2. Войдите под пользователем
3. F12 → Console → введите:
   ```javascript
   fetch("/api/check-session").then(r => r.json()).then(console.log)
   ```

**Ожидается:** `{"username": "Ваше имя", ...}`

**Если `username: null`:** Сессия не сохранилась — очистите кук/и.

---

### Шаг 3: Проверка сохранения

В консоли (F12):
```javascript
fetch("/api/tree", {
    method: "POST",
    headers: {"Content-Type": "application/json"},
    credentials: "same-origin",
    body: JSON.stringify({
        persons: {"p1": {"id": "p1", "name": "Тест"}},
        marriages: [],
        current_center: "p1"
    })
}).then(r => r.json()).then(console.log)
```

**Ожидается:** `{"ok": true}`

**Если ошибка:** Смотрите текст ошибки в консоли.

## ✅ БЫСТРОЕ РЕШЕНИЕ

1. **Очистите кэш и кук/и:**
   - Chrome/Edge: `Ctrl+Shift+Delete` → Clear browsing data
   - Firefox: `Ctrl+Shift+Delete` → Clear Data

2. **Попробуйте режим инкогнито:**
   - Chrome: `Ctrl+Shift+N`
   - Firefox: `Ctrl+Shift+P`

3. **Отключите расширения:**
   - uBlock Origin
   - AdBlock
   - Privacy Badger

4. **Проверьте с другого браузера:**
   - Если работает в Chrome но не в Firefox — проблема в настройках Firefox

## 🔧 ИЗМЕНЕНИЯ В ПРИЛОЖЕНИИ

Если проблема в CORS, добавьте в `web/app.py`:

```python
from flask_cors import CORS

# После создания app
CORS(app, supports_credentials=True, origins=["*"])
```

**Или установите переменную на Railway:**
```
FLASK_ENV=development
```
