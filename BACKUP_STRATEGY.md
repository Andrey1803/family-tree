# 🛡️ СИСТЕМА ЗАЩИТЫ ДАННЫХ

## 📊 УРОВНИ ЗАЩИТЫ

### **Уровень 1: localStorage браузера** ✅
**Где:** Браузер пользователя
**Что:** Автоматическое сохранение каждые 5 минут
**Как:** `localStorage.setItem('family_tree_backup', JSON.stringify(treeData))`

**Плюсы:**
- ✅ Мгновенное сохранение
- ✅ Работает без интернета
- ✅ Нет задержек

**Минусы:**
- ❌ Данные только в одном браузере
- ❌ Пропадают при очистке кэша
- ❌ Нет доступа с других устройств

---

### **Уровень 2: Сервер синхронизации** ✅
**Где:** Railway (`ravishing-caring`)
**Что:** SQLite база данных
**Как:** POST `/api/sync/upload`

**Плюсы:**
- ✅ Доступ с любого устройства
- ✅ Данные не пропадут при очистке браузера
- ✅ Автоматическая синхронизация

**Минусы:**
- ❌ Нужен интернет
- ❌ Зависит от Railway

---

### **Уровень 3: ZIP-экспорт** ✅
**Где:** Файл на компьютере пользователя
**Что:** `backup_username_YYYYMMDD_HHMMSS.zip`
**Как:** GET `/api/backup/export`

**Плюсы:**
- ✅ Полная копия данных
- ✅ Можно хранить где угодно
- ✅ Независимо от сервера

**Минусы:**
- ❌ Нужно вручную скачивать
- ❌ Может устареть

---

### **Уровень 4: ZIP-импорт** 🆕
**Где:** Загрузка файла пользователем
**Что:** Восстановление из ZIP
**Как:** POST `/api/backup/import`

**Плюсы:**
- ✅ Восстановление из любой копии
- ✅ Перенос между аккаунтами
- ✅ Слияние деревьев

**Минусы:**
- ❌ Нужно вручную загружать

---

### **Уровень 5: История версий** 🆕
**Где:** Папка `backups/username/`
**Что:** До 10 последних версий
**Как:** Автоматически при сохранении

**Плюсы:**
- ✅ Можно откатить к старой версии
- ✅ Защита от случайных удалений
- ✅ Хранение на сервере

**Минусы:**
- ❌ Занимает место на сервере

---

## 🔧 КАК ИСПОЛЬЗОВАТЬ

### **Для пользователей:**

#### **1. Автоматическое сохранение**
```
Просто работайте в приложении!
Данные сохраняются каждые 5 минут автоматически.
```

#### **2. Ручное сохранение**
```
Меню → Файл → Сохранить
Или Ctrl+S
```

#### **3. Экспорт (скачать копию)**
```
Меню → Файл → Экспорт → Скачать ZIP
Сохраните файл в надёжное место!
```

#### **4. Импорт (восстановить)**
```
Меню → Файл → Импорт → Загрузить ZIP
Выберите файл backup_*.zip
```

#### **5. История версий**
```
Меню → Файл → История версий
Выберите версию для восстановления
```

---

### **Для разработчиков:**

#### **Добавить экспорт в UI:**

```html
<!-- tree.html -->
<button onclick="exportTree()">📥 Скачать копию</button>

<script>
async function exportTree() {
    const response = await fetch('/api/backup/export');
    const blob = await response.blob();
    const url = window.URL.createObjectURL(blob);
    const a = document.createElement('a');
    a.href = url;
    a.download = `backup_${username}_${Date.now()}.zip`;
    a.click();
}
</script>
```

#### **Добавить импорт в UI:**

```html
<!-- tree.html -->
<input type="file" id="import-file" accept=".zip" style="display:none">
<button onclick="document.getElementById('import-file').click()">📤 Загрузить копию</button>

<script>
document.getElementById('import-file').addEventListener('change', async (e) => {
    const file = e.target.files[0];
    if (!file) return;
    
    const formData = new FormData();
    formData.append('backup', file);
    
    const response = await fetch('/api/backup/import', {
        method: 'POST',
        body: formData
    });
    
    const result = await response.json();
    if (result.ok) {
        alert('✅ Дерево восстановлено!');
        location.reload();
    } else {
        alert('❌ Ошибка: ' + result.error);
    }
});
</script>
```
