# 🔧 СРОЧНО: Выполнить миграцию на Railway

## Проблема
База данных на сервере не имеет колонки `photo_full`, что вызывает ошибку 500 при загрузке дерева.

**Ошибка:** `table persons has no column named photo_full`

---

## Решение: Выполнить SQL через консоль Railway

### Шаг 1: Откройте консоль в Railway Dashboard

1. Перейдите в **Railway Dashboard**: https://railway.app/
2. Выберите проект `family-tree-production`
3. Выберите сервис `ravishing-caring`
4. Нажмите **Console** (или New → Console)

### Шаг 2: Выполните миграцию

В консоли выполните:

```bash
sqlite3 /data/family_tree.db "ALTER TABLE persons ADD COLUMN photo_full BLOB;"
```

Или, если хотите проверить сначала:

```bash
# Проверка структуры
sqlite3 /data/family_tree.db "PRAGMA table_info(persons);"

# Добавление колонки
sqlite3 /data/family_tree.db "ALTER TABLE persons ADD COLUMN photo_full BLOB;"

# Проверка результата
sqlite3 /data/family_tree.db "PRAGMA table_info(persons);"
```

**Ожидаемый результат:**
```
...
33|photo_full|BLOB|0|0|0
...
```

### Шаг 3: Проверьте загрузку дерева

После выполнения миграции попробуйте загрузить дерево снова:

```powershell
python test_upload_debug.py
```

Или через веб-приложение.

---

## Альтернатива: Через Python-скрипт

Если SQLite недоступен, выполните Python-скрипт в консоли Railway:

```bash
# В консоли Railway
cd /app
python << 'EOF'
import sqlite3
conn = sqlite3.connect('/data/family_tree.db')
cursor = conn.cursor()

# Проверяем колонки
cursor.execute("PRAGMA table_info(persons)")
columns = [row[1] for row in cursor.fetchall()]

if 'photo_full' not in columns:
    print("Adding photo_full column...")
    cursor.execute("ALTER TABLE persons ADD COLUMN photo_full BLOB")
    conn.commit()
    print("Done!")
else:
    print("photo_full already exists")

conn.close()
EOF
```

## Проверка

После миграции:

1. ✅ `/api/health` возвращает `status: ok`
2. ✅ Загрузка дерева работает (нет ошибки 500)
3. ✅ Скачивание дерева возвращает 44 персоны и 14 браков

---

## Почему это произошло?

`prestart.sh` выполняется **до** запуска gunicorn, но в старой версии базы данных уже была таблица `persons` без `photo_full`. Миграция должна была сработать, но возможно:

1. БД была создана до добавления миграции в `prestart.sh`
2. Миграция не выполнилась из-за ошибки
3. Volume `/data` был примонтирован позже

**Решение:** Выполнить миграцию вручную.
