# 🔴 ПРОБЛЕМЫ DESKTOP-ПРИЛОЖЕНИЯ (на 02.03.2026)

## Проблема 1: Палитра цветов не сохраняется между запусками

**Симптомы:**
- Пользователь выбирает цвета в "Сервис → Цвет интерфейса"
- Нажимает "Применить" — цвета применяются
- Закрывает приложение, запускает снова — цвета сбросились на белые/по умолчанию

**Что сделано:**
- `PALETTE_FILE = os.path.join("data", "palette.json")` — относительный путь
- `save_palette_to_file()` вызывается при применении цветов
- Добавлено логирование сохранения/загрузки
- `main.py` устанавливает рабочую директорию в корень проекта/.exe папку

**Возможная причина:**
- Путь к `palette.json` вычисляется неправильно при запуске .exe
- Файл сохраняется в одну папку, загружается из другой

**Файлы:**
- `Дерево/constants.py` — `PALETTE_FILE`, `load_palette_from_file()`, `save_palette_to_file()`
- `main.py` — установка рабочей директории
- `Дерево/app.py` — `open_color_palette_dialog()`, `apply_ui_colors_from_palette()`

## Проблема 3: Дерево стартует пустым, рисуется с задержкой

**Симптомы:**
- При запуске показывается окно приветствия
- Холст пустой (белый/серый)
- Через 2-5 секунд дерево появляется

**Что сделано:**
- Добавлен `self.root.update_idletasks()` перед `refresh_view()`

**Возможная причина:**
- `refresh_view()` не вызывается сразу
- Данные загружаются асинхронно
- `update_idletasks()` недостаточно для немедленной перерисовки

**Файлы:**
- `Дерево/app.py` — `__init__()`, `refresh_view()`
- `Дерево/models.py` — `load_from_file()`
