# 🔧 ИСПРАВЛЕНИЕ: Сайт не сохраняет данные на сервере

## ❌ ПРОБЛЕМА

Веб-приложение на Railway не сохраняет данные (деревья, пользователей, фото) потому что **не настроено хранилище (volume)**.

---

## 🔍 ПРИЧИНА

На Railway по умолчанию файловая система **временная**. При каждом деплое все данные в папке `/data` удаляются.

Для постоянного хранения нужно создать **Volume** (постоянное хранилище).

---

## ✅ РЕШЕНИЕ

### Шаг 1: Создать Volume на Railway

1. Откройте [Railway Dashboard](https://railway.app/)
2. Перейдите в проект `family-tree-production`
3. Выберите сервис `family-tree` (веб-приложение)
4. Нажмите вкладку **"Volumes"**
5. Нажмите **"+ New Volume"**
6. Настройте:
   - **Mount Path:** `/data`
   - **Size:** 1 GB (минимум)
7. Нажмите **"Create"**

### Шаг 2: Проверить переменные окружения

В Railway Dashboard → `family-tree` → **Variables**:

```bash
DATA_DIR=/data
SECRET_KEY=your-secret-key-here
SYNC_SERVER_URL=https://ravishing-caring-production-3656.up.railway.app
PORT=5000
```

**Важно:** Убедитесь, что `DATA_DIR=/data` установлен!

### Шаг 3: Перезапустить сервис

1. В Railway Dashboard → `family-tree` → **Settings**
2. Нажмите **"Restart"**
3. Дождитесь перезапуска (1-2 минуты)

### Шаг 4: Проверить логи

В Railway Dashboard → `family-tree` → **Deployments** → **View Logs**

Должны появиться сообщения:
```
[CONFIG] Using DATA_DIR from env: /data
[CONFIG] USERS_FILE=/data/users.json
[TREE_SERVICE] Loaded X persons for username
```

В Railway Dashboard → `family-tree` → **Deployments** → **Console**:

```bash

```

## 🔄 АЛЬТЕРНАТИВНОЕ РЕШЕНИЕ (без Volume)

Если Volume создать нельзя, можно использовать **базу данных SQLite** вместо JSON файлов.

### Изменения в коде:

1. **web/app.py** - изменить `_data_dir` на временную папку
2. **Добавить синхронизацию** с сервером синхронизации при каждом сохранении

**Но это временное решение!** Данные всё равно будут теряться при деплое.

## 📞 ЕСЛИ НЕ РАБОТАЕТ

### Проверьте:

1. **Volume создан?**
   ```bash
   # В консоли Railway
   df -h /data
   ```

2. **Переменные окружения установлены?**
   ```bash
   # В консоли Railway
   echo $DATA_DIR
   ```

3. **Права на запись?**
   ```bash
   # В консоли Railway
   touch /data/test.txt
   ls -la /data/
   ```

4. **Логи приложения:**
   - Откройте Railway Dashboard
   - `family-tree` → **Deployments** → **View Logs**
   - Ищите ошибки `[CONFIG]` или `[TREE_SERVICE]`
