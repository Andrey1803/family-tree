## 🔒 Критические исправления безопасности

### Уязвимость изоляции данных пользователей (Март 2026)

**Проблема:** При входе обычного пользователя после администратора в той же сессии браузера, новый пользователь мог видеть данные дерева администратора (все персоны).

**Причина:** `server_token` администратора не очищался при входе нового пользователя.

**Исправление:** В функциях `login()` и `register()` добавлена очистка `server_token` перед входом нового пользователя:

```python
# ВАЖНО: Очищаем старый server_token перед входом нового пользователя!
session.pop('server_token', None)
session.pop('server_user_id', None)
```

**Файл:** `web/app.py`

**Затронутые функции:**
- `login()` - очистка при входе
- `register()` - очистка при регистрации
- `logout()` - полная очистка сессии (уже было)

**Тестирование:**
1. Войдите как администратор
2. Выйдите через /logout
3. Войдите как новый пользователь
4. **Результат:** Должно загрузиться пустое дерево нового пользователя
