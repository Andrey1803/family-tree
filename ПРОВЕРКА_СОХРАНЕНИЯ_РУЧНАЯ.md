## 🔍 ПРОВЕРКА ЧЕРЕЗ RAILWAY CONSOLE

### Для веб-приложения `family-tree`:

1. Откройте Railway Dashboard
2. `family-tree` → **Deployments** → **Console**
3. Выполните команды:

```bash
# Проверить файлы данных
ls -la /data/

# Проверить содержимое users.json
cat /data/users.json

# Проверить дерево пользователя
cat "/data/family_tree_Тестовый Пользователь.json"
```

**Должны появиться файлы:**
```
/data/
├── users.json
├── family_tree_Тестовый Пользователь.json
└── palette.json (если меняли цвета)
```

## 🐛 ВОЗМОЖНЫЕ ПРОБЛЕМЫ

### 1. Данные не сохраняются после перезапуска

**Причина:** Volume не подключён или DATA_DIR неверный

**Решение:**
- Проверьте в Railway: `DATA_DIR = /data`
- Проверьте что Volume создан (5 GB)
- Перезапустите сервис

### 2. Ошибка при сохранении

**Проверьте логи:**
- Railway Dashboard → `family-tree` → **Deployments** → **View Logs**
- Ищите ошибки `[TREE_SERVICE]` или `[API_TREE]`

### 3. Пользователь не может войти

**Проверьте users.json:**
```bash
cat /data/users.json
```

Должен быть формат:
```json
{
  "users": {
    "Тестовый Пользователь": "хэш_пароля"
  }
}
```
